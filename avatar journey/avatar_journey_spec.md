# Avatar Journey - 虚拟旅行游戏开发规格文档

## 项目概述
Avatar Journey 是一个基于 AI 的虚拟旅行游戏，让用户创建自己的化身进行实时旅行体验。游戏按真实时间运行，AI 会定期生成旅行事件和体验。

## 核心功能模块

### 1. 角色设定模块
- **功能**：用户输入主人公的设定提示词
- **输入内容**：
  - 角色姓名
  - 性格描述
  - 兴趣爱好
  - 背景故事
  - 其他个性化特征
- **存储**：保存为角色档案，供 Claude API 调用

### 2. 旅行配置模块
- **出发地和目的地**：
  - 出发地选择（类似买火车票的起点）
  - 目的地选择（类似买火车票的终点）
  - 可以是城市、国家或具体地点
  - 支持预设选项和自定义输入
- **旅行方式选择**：
  - 自驾游
  - 徒步旅行
  - 公共交通
  - 背包客
  - 豪华旅行
- **旅行风格**：
  - 探险型
  - 休闲型
  - 文化体验型
  - 美食寻访型

### 3. 实时旅行引擎
#### 3.1 时间系统
- **运行时间**：每天 6:00-22:00（16小时活跃期）
- **睡眠时间**：22:00-6:00（8小时休息期）
- **更新频率**：非固定间隔，由 AI 判断下次事件时间
- **时间加速**：用户离线时可适度加速无关紧要的时段

#### 3.2 事件生成机制
- **AI 任务循环**：
  1. 分析当前状态和上一个任务
  2. 生成下一个事件内容
  3. 估算事件持续时间
  4. 设定下次更新的具体时间点
  5. 等待时间到达后执行下次循环

#### 3.3 事件类型
- **日常活动**：吃饭、住宿、交通、观光
- **随机遇遇**：遇到陌生人、发现有趣地点
- **天气影响**：AI 生成天气变化影响行程
- **当地活动**：节庆、集市、文化活动（AI 根据目的地知识生成）
- **小意外**：错过交通、迷路、意外发现等

### 4. 用户交互系统
#### 4.1 实时干预功能
- **触发时机**：当 AI 生成需要选择的情况时
- **交互方式**：
  - 用户可发送消息给虚拟角色
  - 影响角色的决策和行动
  - 例如：建议与某人深入交流、选择不同路线等
- **响应机制**：AI 根据用户建议调整后续行程

#### 4.2 被动观察模式
- **自动更新**：无用户干预时，AI 自主推进剧情
- **消息推送**：定时向用户发送旅行更新
- **状态展示**：实时显示角色当前位置和活动

### 5. 旅行日记系统
#### 5.1 日记生成
- **频率**：每天生成一篇日记
- **内容**：当日主要事件总结（约200字）
- **格式**：日期 + 标题 + 内容段落
- **AI 调用**：使用独立的 Claude API 调用生成日记

#### 5.2 记忆管理
- **长期记忆**：通过日记系统维持角色记忆
- **上下文传递**：新的 API 调用时包含相关历史日记
- **存储优化**：避免传递过长的对话历史

## 技术架构

### 前端技术栈
- **框架**：React
- **UI 组件**：使用 Tailwind CSS 或类似样式方案
- **状态管理**：React useState/useContext
- **实时更新**：轮询或 WebSocket 连接后端

### 后端技术栈
- **服务器**：Node.js + Express
- **定时任务**：node-cron 或类似定时任务库
- **数据存储**：
  - 简单方案：JSON 文件存储
  - 扩展方案：SQLite 或 MongoDB
- **API 集成**：Claude API 调用模块

### 数据结构设计

#### 角色数据
```json
{
  "characterId": "uuid",
  "name": "角色名",
  "description": "角色设定提示词",
  "travelStyle": "旅行方式",
  "destination": "目的地",
  "currentStatus": {
    "location": "当前位置",
    "activity": "当前活动",
    "lastUpdate": "最后更新时间"
  }
}
```

#### 旅行事件数据
```json
{
  "eventId": "uuid",
  "characterId": "角色ID",
  "timestamp": "事件时间",
  "type": "事件类型",
  "content": "事件内容",
  "needsUserInput": true/false,
  "nextEventTime": "下次事件预计时间"
}
```

#### 日记数据
```json
{
  "diaryId": "uuid",
  "characterId": "角色ID", 
  "date": "日期",
  "title": "日记标题",
  "content": "日记内容",
  "keyEvents": ["重要事件列表"]
}
```

## Claude API 集成方案

### API 调用场景
1. **事件生成调用**：生成下一个旅行事件
2. **用户交互调用**：处理用户干预时的响应
3. **日记生成调用**：每日总结旅行经历

### Prompt 设计原则
- **状态保持**：每次调用包含完整的角色设定和相关历史
- **真实性约束**：强调生成合理、真实的旅行体验
- **时间感知**：让 AI 理解当前时间和旅行进度
- **开放性平衡**：既要有随机性，又要保持逻辑连贯

### 错误处理机制
- **API 调用失败**：重试机制 + 降级处理
- **响应解析错误**：默认响应模板
- **定时器异常**：系统重启时的状态恢复

## 用户界面设计要求

### 主要页面
1. **角色创建页**：设定角色和旅行参数
2. **旅行进行页**：实时显示旅行状态和事件
3. **交互对话页**：与角色进行文字交流
4. **日记查看页**：浏览历史旅行日记

### 交互设计原则
- **简洁直观**：避免复杂操作
- **实时反馈**：及时显示系统状态
- **沉浸体验**：营造真实的旅行感受

## 部署和运行

### 环境要求
- **Node.js**：版本 18+
- **包管理**：npm 或 yarn
- **API Key**：Claude API 密钥配置

### 配置项
- Claude API 调用频率限制
- 定时任务调度设置
- 数据存储路径配置
- 前端服务器端口设置
